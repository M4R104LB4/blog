Preparing my Wardriving Setup for 2026

Intro

The things didn't started as I was expecting, so lets introduce this post talking about failure. I started working on my setup for BSIDES with the idea of portability so I used a Raspberry PI. It seemed to have all what I needed except one thing, It was missing a screen, so I added an LCD screen. Of course, it was not a very good brand so the driver increased its processing to the hell and basically it didn't have the performance I was expecting to face the BSIDES challenge. It was hard to abbandom this idea because of the time invested but I learned and made the screen work which was not easy.

So, my second choice in my Gadget vault was a Hack5 MARK VII. It was a new device for me so here is the configuration I made on it.

The first thing to do was watching all the videos of this crazy guy of Hak5, I took some of the configuration steps from the videos.

GPS INSTALLATION

The GPS that I used was recommended as one of the best modules for wardriving in one of the videos, it is a G-MOUSE VK-172. In order to install the module drivers in the Mark VII (OS OpenWRT) I executed the following commands. 

opkg update
opkg install gpsd gpsd-clients

Once installed, the next step was verifying It was sending data correctly with those commands:

dmesg | grep ttyACM0 
cat /dev/ttyACM0

The next command is an extra step which is needed because the GPS module I am using. Kismet does not support binary format in the data which is being sent by the VK-172, it supports NMEA, so "gpsd" is in charged of this conversion.

gpsd /dev/ttyACM0

Finally, I verified that the GPS was receiving correctly signals from some satelites with the following command:

cgps

HUB MODIFICATION TO ADAPT A USB EXTERNAL BATTERY

As I had beed advised in 2025, some wardriving components need a good power supply to increase the reception of the signals and it included the GPS module which was having troubles to get a stable GPS signal with the supply of the usb port. So I decided to modify an cheap USB HUB that I owned. The modification was basically soldering the USB connector directly to a DC connector in the PCB, to connect an external battery as power supply. I know it is not elegant but it works.

ALFA NETWORK INSTALLATION

After setting up the GPS module and the HUB, my next concern was the ALFA networks card, It was not necessary but I included it to cover the 5 GHZ band. The command that I used to install it was generated by an AI and It worked well.

opkg install kmod-rtl8812au-ct

Finally, to set the ALFA in monitor mode I used the following commands: 

wifi down
ip link set wlan3 down
iw wlan3 set monitor control
ip link set wlan3 up

To ensure everything had been installed correctly and the ALFA was working in the band of 5 Ghz I executed the following commands: 

iw dev
iw list

KISMET CONFIGURATION

I left Kismet as the final step. I used this page which I considered as interesting in 2025 when I participated by first time in the Wardrining contest of BSIDES. The link of the video is here:

https://www.tdtlatinoamerica.com.ar/forotdt/viewtopic.php?t=3359

The command needed to install kismet in the Mark VII is this:

opkg install kismet kismet-capture-linux-wifi

I have also added a couple of congigurations based on the site of tdlatinoamérica and the AI.

gps=gpsd:host=localhost,port=2947

source=wlan1:type=linuxwifi,name=MARK7,hop=true,band=abg
source=wlan3:type=linuxwifi,name=ALFA,hop=true,band=abg

I used the command "kismet &" in order TO keep the Kismet proces persistant, the hardware performed nice. I know it is a flag to run this process daemonized but I am still testing.
